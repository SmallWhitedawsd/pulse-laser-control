# 脉冲间隙控制器 — 实现方案（v2：FIFO 环形缓冲架构）

> 基于 STM32F103C8T6 · 生产者-消费者解耦 · 首脉冲直通旁路

---

## 1. 需求摘要

| 项目 | 规格 |
|------|------|
| 输入波形 | 0~3.3V TTL，1~N 个周期的脉冲串，频率 1KHz~100KHz |
| 输出波形 | 0~3.3V TTL，N 个脉冲进 → N 个脉冲出，脉冲高电平宽度 = 原始输入脉宽 |
| 可控参数 | **输出相邻脉冲之间的低电平间隙**，可调范围 1s~50s，精度 1ms |
| 实时性 | 首脉冲延迟 ≤5 个输入周期（100KHz 下即 ≤50μs） |
| 参数设置 | 上位机通过 UART 串口下发间隙参数 |
| 架构策略 | **FIFO 环形缓冲（蓄水池）**：生产者（输入捕获）与消费者（输出控制）完全解耦 |

---

## 2. 架构总览

### 2.1 核心理念

> 把 STM32 想象成一个水库：
> - **TIM2 输入捕获** = 进水管上的流量计，只负责疯狂记录每次进了多少水（脉宽），压入 FIFO
> - **主循环 + TIM3/TIM4** = 出水阀，严格按照设定：放一波水（输出脉冲）→ 关紧阀门休息指定秒数 → 再放下一波

这样彻底满足「不漏过任何一个脉冲，且间距严格可控」。

## 3. 硬件方案

### 3.1 主控芯片

**STM32F103C8T6**（Cortex-M3，72MHz，64KB Flash / 20KB SRAM）

选用理由：
- 72MHz 主频，中断响应 ~12 周期 ≈ 0.17μs
- 在 100KHz（周期 10μs）输入下，ISR 只需几十个机器周期即可退出，完全跟得上
- 20KB SRAM 足够容纳 1024 深度环形队列（仅占 2KB）
- 4 个通用定时器 + 2 个高级定时器，资源充足

### 3.2 引脚分配

| 功能 | GPIO | 外设 | 说明 |
|------|------|------|------|
| 脉冲输入 | PA0 | TIM2_CH1（输入捕获） | 双沿捕获 |
| 脉冲输出 | PA1 | GPIO 直驱 | BSRR 寄存器直接翻转 |
| 调试串口 TX | PA9 | USART1_TX | 接上位机 RX |
| 调试串口 RX | PA10 | USART1_RX | 接上位机 TX |
| 状态 LED | PC13 | GPIO（板载 LED） | 指示工作状态 |
| SWDIO | PA13 | SWD | 调试/烧录 |
| SWCLK | PA14 | SWD | 调试/烧录 |

### 3.3 定时器分配

| 定时器 | 用途 | 配置 | 分辨率 |
|--------|------|------|--------|
| **TIM2** | 输入捕获（生产者） | CH1 双沿捕获，72MHz 时钟，PSC=0 | 13.9ns |
| **TIM4** | 脉冲宽度输出定时器（消费者） | 1MHz 时基，PSC=72-1, ARR=65535 | **1μs** |
| **TIM3** | 间隙等待定时器（消费者） | 1ms 时基，PSC=72000-1, ARR=1000-1 | **1ms** |

## 4. 核心数据结构：环形队列

利用 2 的 N 次方长度 + 位与运算 `&` 代替取模 `%`，榨干 CPU 性能。在 100KHz 中断频率下这一点至关重要。

```c
#define FIFO_SIZE  1024          // 必须是 2 的 N 次方（1024 个脉冲 × 2 字节 = 2KB）
#define FIFO_MASK  (FIFO_SIZE - 1)

volatile uint16_t pulse_fifo[FIFO_SIZE];  // 环形缓冲区
volatile uint16_t fifo_head = 0;          // 写入指针（生产者更新）
volatile uint16_t fifo_tail = 0;          // 读取指针（消费者更新）

#define FIFO_IS_EMPTY()   (fifo_head == fifo_tail)
#define FIFO_IS_FULL()    (((fifo_head + 1) & FIFO_MASK) == fifo_tail)

// 入队（生产者调用，ISR 中）
static inline void fifo_push(uint16_t width) {
    pulse_fifo[fifo_head] = width;
    fifo_head = (fifo_head + 1) & FIFO_MASK;
}

// 出队（消费者调用，主循环中）
static inline uint16_t fifo_pop(void) {
    uint16_t width = pulse_fifo[fifo_tail];
    fifo_tail = (fifo_tail + 1) & FIFO_MASK;
    return width;
}
```

> **容量分析**：最坏情况下输入 100KHz 连续脉冲串，每秒产生 100,000 个宽度数据。但实际受限于 1s~50s 死区间隙，在死区期间脉冲不被输出但照常入队，所以 FIFO 深度 1024 在 100KHz 输入下仅能缓冲约 10ms 的脉冲。对于大多数脉冲串场景（几十到几百个脉冲）绰绰有余。若需更长的缓冲，可将 `FIFO_SIZE` 增大到 4096（8KB）或 8192（16KB）。

## 5. 生产者：TIM2 输入捕获 ISR

中断里的逻辑极度精简 —— **不要在这里直接控制 PA1 输出**，只做一件事：算脉宽，入队。唯一例外是「首脉冲直通旁路」。

### 5.1 基本逻辑

```c
volatile uint16_t capture_start = 0;
volatile uint8_t  first_pulse_bypassed = 0;  // 旁路标记

void TIM2_IRQHandler(void) {
    if (TIM2->SR & TIM_SR_CC1IF) {
        uint16_t capture = TIM2->CCR1;
        uint8_t  edge    = (TIM2->CCER & TIM_CCER_CC1P) ? FALLING : RISING;

        if (edge == RISING) {
            // ── 上升沿 ──
            capture_start = capture;

            // 【直通旁路】：如果消费端空闲且队列为空，首个脉冲直接穿透到输出
            if (g_out_state == IDLE && FIFO_IS_EMPTY()) {
                GPIOA->BSRR = GPIO_BSRR_BS1;   // PA1 立即拉高（寄存器直写）
                g_out_state = PULSE_OUT;        // 强制消费端状态机
                first_pulse_bypassed = 1;       // 标记此脉冲已提前输出
            }
            // 否则：什么都不做，等下降沿来了一起算脉宽入队

        } else {
            // ── 下降沿 ──
            uint16_t width = capture - capture_start;

            if (first_pulse_bypassed) {
                // 已经直通输出的脉冲：直接拉低 PA1，进入间隙等待
                GPIOA->BSRR = GPIO_BSRR_BR1;
                deadtime_start(g_gap_ms);
                g_out_state = GAP_WAIT;
                first_pulse_bypassed = 0;
                // 不写入 FIFO（这个脉冲已经被消费了）
            } else {
                // 普通路径：脉宽入队
                if (!FIFO_IS_FULL()) {
                    fifo_push(width);
                }
                // FIFO 满则丢弃并计数溢出（调试用）
            }
        }

        // 切换捕获边沿
        TIM2->CCER ^= TIM_CCER_CC1P;
    }
}
```

```
输入 PA0:  ──┐   ┌──┐   ┌──┐   ┌──┐   ┌─────────────
             │   │  │   │  │   │  │   │
             └───┘  └─┘   └─┘   └─┘   └─
              │  │        │  │        │
              │  └─width1─┘  └─width2─┘
              ▼              ▼
        ┌──────────────────────────┐
FIFO:   │ width1 │ width2 │  ...  │  (纯数据，不包含时序)
        └──────────────────────────┘
```

## 6. 消费者：输出状态机

### 6.1 状态定义

```
                    ┌──────────┐
         系统启动 ──▶│   IDLE   │◀────────────────────────────┐
                    └────┬─────┘                              │
                         │ FIFO 非空 或 首脉冲直通触发           │
                         ▼                                    │
                    ┌────────────┐                             │
                    │ PULSE_OUT  │   PA1=高，TIM4 计时 width μs │
                    └─────┬──────┘                             │
                          │ TIM4 到期（脉宽结束）               │
                          ▼                                    │
                    ┌────────────┐                             │
                    │  GAP_WAIT  │   PA1=低，TIM3 计时 gap_ms  │
                    └─────┬──────┘                             │
                          │ TIM3 到期（间隙结束）                │
                          └────────────────────────────────────┘
```

| 状态 | PA1 电平 | 条件 | 动作 |
|------|---------|------|------|
| **IDLE** | 低 | 等待数据 | 检测 FIFO 非空 → 出队，置高 PA1，启动 TIM4，进入 PULSE_OUT |
| **PULSE_OUT** | 高 | TIM4 计时中 | TIM4 到期 → 置低 PA1，启动 TIM3，进入 GAP_WAIT |
| **GAP_WAIT** | 低 | TIM3 计时中 | TIM3 到期 → 进入 IDLE（准备打出下一个脉冲） |

### 6.2 实现代码

消费者放在主循环中轮询，不依赖复杂中断嵌套：

```c
typedef enum { IDLE, PULSE_OUT, GAP_WAIT } OutState;
volatile OutState g_out_state = IDLE;

volatile uint8_t timer4_expired = 0;      // TIM4 到期标志
volatile uint8_t first_pulse_bypassed = 0; // 生产者设置的旁路标记

void process_pulse_output(void) {
    switch (g_out_state) {

        case IDLE:
            if (!FIFO_IS_EMPTY()) {
                // 1. 出队一个脉宽
                uint16_t current_width = fifo_pop();

                // 2. PA1 置高
                GPIOA->BSRR = GPIO_BSRR_BS1;

                // 3. 启动 TIM4，精确保持高电平 current_width 微秒
                start_timer4_us(current_width);
                g_out_state = PULSE_OUT;
            }
            break;

        case PULSE_OUT:
            if (timer4_expired) {
                timer4_expired = 0;

                // 1. PA1 置低
                GPIOA->BSRR = GPIO_BSRR_BR1;

                // 2. 启动长间隙定时器
                deadtime_start(g_gap_ms);
                g_out_state = GAP_WAIT;
            }
            break;

        case GAP_WAIT:
            if (deadtime_remaining_ms == 0) {
                g_out_state = IDLE;
            }
            break;
    }
}
```

### 6.3 消费者时序示意

```
FIFO 队列:  ...[width3][width2][width1] → 出队方向

PA1 输出:   ────────┐           ┌──────────┐           ┌──────
                    │           │          │           │
                    └───────────┘          └───────────┘
                    │  width1   │          │  width2   │
                    │           │◄─ gap ──▶│           │
                    │ PULSE_OUT │ GAP_WAIT │ PULSE_OUT │
```

## 7. 辅助定时器实现

### 7.1 TIM4：脉宽输出定时器（1μs 分辨率）

```c
// 初始化：PSC = 72-1 → 1MHz (1μs), ARR = 65535（最大 65.535ms 脉宽）
void init_timer4(void) {
    RCC->APB1ENR |= RCC_APB1ENR_TIM4EN;
    TIM4->PSC = 72 - 1;           // 1MHz
    TIM4->ARR = 65535;            // 最大周期
    TIM4->DIER |= TIM_DIER_UIE;   // 使能溢出中断
    NVIC_EnableIRQ(TIM4_IRQn);
    // 不启动，由消费者按需启动
}

void start_timer4_us(uint16_t width_us) {
    TIM4->CNT = 0;
    TIM4->ARR = width_us;          // 精确到 1μs
    TIM4->SR = ~TIM_SR_UIF;       // 清标志
    TIM4->CR1 |= TIM_CR1_CEN;     // 启动
}

void TIM4_IRQHandler(void) {
    if (TIM4->SR & TIM_SR_UIF) {
        TIM4->SR = ~TIM_SR_UIF;
        TIM4->CR1 &= ~TIM_CR1_CEN; // 停止
        timer4_expired = 1;        // 通知消费者
    }
}
```

> **脉宽范围**：TIM4 为 16 位，最大脉宽 = 65535μs ≈ 65.5ms。对于 100KHz 输入（周期 10μs），脉冲宽度远在此范围内，绰绰有余。

### 7.2 TIM3：间隙定时器（1ms 分辨率）

```c
volatile uint32_t deadtime_remaining_ms = 0;

void init_timer2(void) {
    RCC->APB1ENR |= RCC_APB1ENR_TIM3EN;
    TIM3->PSC = 72000 - 1;        // 1KHz → 1ms 周期
    TIM3->ARR = 1000 - 1;         // 1ms 溢出一次（冗余，用软件计数决定总时长）
    TIM3->DIER |= TIM_DIER_UIE;
    NVIC_EnableIRQ(TIM3_IRQn);
    // 不启动
}

void deadtime_start(uint32_t gap_ms) {
    deadtime_remaining_ms = gap_ms;
    TIM3->CNT = 0;
    TIM3->SR = ~TIM_SR_UIF;
    TIM3->CR1 |= TIM_CR1_CEN;
}

void TIM3_IRQHandler(void) {
    if (TIM3->SR TIM2->SR & TIM_SR_UIF TIM_SR_UIF) {
        TIM3->SR = ~TIM_SR_UIF;
        if (--deadtime_remaining_ms == 0) {
            TIM3->CR1 TIM2->CR1 &= ~TIM_CR1_CEN;= ~TIM_CR1_CEN;  // 停止
        }
    }
}
```

> 1ms 精度完全满足要求（50s 最大间隙下误差仅 1ms / 50000ms = 0.002%）。

## 8. 中断优先级

| 中断源 | 优先级 (NVIC) | 说明 |
|--------|---------------|------|
| **TIM2 输入捕获** | **最高**（Preempt 0, Sub 0） | 100KHz 下不能丢边沿，ISR 必须极致精简 |
| TIM4 脉宽定时器 | 中（Preempt 1, Sub 0） | ⩽65ms 一次 |
| TIM3 间隙定时器 | 中（Preempt 1, Sub 1） | 每 1ms 一次 |
| USART1 RX | 低（Preempt 2, Sub 0） | 无实时要求 |

## 9. 首脉冲延迟分析

### 9.1 直通旁路路径（系统 IDLE 时的第一个脉冲）

```
输入上升沿 (PA0)
    │
    ▼
TIM2 输入捕获触发 → ISR 入口
    │  NVIC 中断延迟: ~12 CPU 周期 ≈ 0.17μs
    ▼
ISR 内判断: g_out_state == IDLE && FIFO 为空
    │  ~3 指令 ≈ 0.04μs
    ▼
GPIOA->BSRR = GPIO_BSRR_BS1  写入 → PA1 翻转为高
    │  1 指令 ≈ 0.014μs
    ▼
输出上升沿 (PA1)
```

**首脉冲总延迟 ≈ 0.25μs**，远低于 100KHz × 5 周期 = 50μs 约束。

### 9.2 普通路径（队列中有数据时）

后续脉冲从 FIFO 出队到输出的延迟取决于主循环轮询周期。在 72MHz 主频下，主循环一次迭代约 1~2μs，加上 TIM4 启动开销，总延迟也在 **几微秒** 量级，依然远低于 50μs。

## 10. 串口通信协议

### 10.1 物理层

- 波特率：**115200**（8N1）

### 10.2 命令集

| 命令 | 功能 | 示例 | 响应 |
|------|------|------|------|

|------|------|

## 12. 主循环结构

```c
int main(void) {
    HAL_Init();
    SystemClock_Config();

    init_gpio();
    init_timer2_capture();    // 生产者：输入捕获
    init_timer4_pulse();      // 消费者：脉宽输出
    init_timer3_gap();        // 消费者：间隙定时器
    init_uart1();             // 串口命令
    load_params();            // 从 Flash 加载间隙参数

    // 启动输入捕获
    TIM3->CR1 |= TIM_CR1_CEN;

    while (1) {
        process_pulse_output();    // 消费者状态机（优先级最高）
        process_uart_command();    // 串口命令解析（非阻塞）
    }
}
```

## 15. 关键设计决策回顾

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 架构 | FIFO 环形缓冲（生产者-消费者） | 解耦输入捕获与输出控制，脉冲不丢失 |
| 队列深度 | 1024（2KB SRAM） | 2 的 N 次方，位与代替取模 |
| 首脉冲延迟 | 直通旁路（Bypass） | IDLE + FIFO 空时 ISR 直接拉 PA1，<0.3μs |
| 输出脉宽精度 | TIM4 @ 1MHz → 1μs | 16 位定时器，最大 65ms 脉宽，足够 |
| 间隙精度 | TIM3 @ 1KHz → 1ms | 50s 最大间隙下误差仅 0.002% |
| 消费者运行位置 | 主循环轮询 | 避免复杂中断嵌套，可维护性好 |

## 16. 与旧方案（v1 死区屏蔽）的对比
